# GitHub Profile Roadmap

## Set Up The Landing Page

1. Create a public GitHub repository with the exact same name as your GitHub username.
2. Add the `README.md` from this folder to the root of that repository.
3. Replace all placeholders, especially `YOUR_USERNAME`, project links, contact links, and tech stack.
4. Pin 3-6 strong repositories or gists on your profile.
5. Add a short bio, location if you want, profile photo, portfolio link, and social links in GitHub profile settings.

## Improve It Over Time

- Keep 3-6 pinned projects current and make sure each has a strong README, screenshots, setup steps, and a clear demo link when possible.
- Make regular contributions through commits, issues, pull requests, and code reviews.
- Publish small but complete projects instead of many unfinished repositories.
- Use GitHub Actions on important repositories to show tests, linting, and deployment checks.
- Add topics, descriptions, licenses, and release notes to public projects.
- Contribute to open source by fixing documentation, reproducing issues, writing tests, or submitting small bug fixes.
- Star useful repositories and organize them into lists so your interests are visible.
- Turn on private contribution visibility if you want your private work to count anonymously on your contribution graph.

## Badges And Achievements To Aim For

- Developer Program Member: join the GitHub Developer Program if you build with the GitHub API or GitHub Apps.
- PRO: appears if you use GitHub Pro.
- Security Bug Bounty Hunter: earned by participating successfully in GitHub's security bug bounty program.
- GitHub Campus Expert: available through the Campus Experts program for eligible students.
- Achievements: GitHub can display achievements for public activity such as notable pull requests, discussions, sponsorship activity, and other community milestones. These change over time, so treat them as a byproduct of useful public work rather than the main goal.

## Monthly Maintenance Checklist

- Update the profile README with your current focus.
- Refresh pinned repositories.
- Add screenshots or demos to one project.
- Open or review at least one useful issue or pull request.
- Archive or hide stale experiments that distract from your best work.
- Write one short technical note in a repo README, GitHub Discussion, or project documentation.
